# To-Do Tracking

**Maintain a `TODO.md` in the project root.**

## Format

```markdown
# TODO — CEL Resource Hub

Last updated: YYYY-MM-DD

## Active (doing now)
- [ ] task — context/blocker

## Up Next
- [ ] task

## Waiting On
- [ ] task — waiting on person/event

## Backlog
- [ ] task

## Done (recent)
- [x] task — date
```

## Rules

1. One file per project, in the root.
2. Update after completing work — check items off, move to Done, add newly discovered tasks.
3. One line per item; details live in plans or session logs.
4. Prune Done to the last ~10; older history is in git and session logs.
5. Read it before starting work; update it after finishing.
