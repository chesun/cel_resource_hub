# Plan — CEL Resource Hub setup

**Date:** 2026-06-21
**Status:** COMPLETED (first pass)

## Goal

Stand up `cel_resource_hub`: a public MkDocs Material site that links every CEL offboard repo
and collects reusable workflow guidance, plus port the useful subset of the research-workflow
infrastructure (logging, ADRs, hooks, agents).

## Approach

1. **Scaffold** a fresh repo in `~/github_repos/cel_resource_hub` (new git history; no research scaffolding).
2. **Site:** `mkdocs.yml` (Material), `requirements.txt`, deploy workflow, `README.md`.
3. **Content (skeleton):** landing page, repository index + one detail page per repo, workflow-tip stubs, resource stubs. Facts derived from each source repo; unknowns marked `TODO`.
4. **Infrastructure:** port hooks (log-reminder, plan-persist, output-length, destructive-action-guard, post-rewrite-verify, protect-files, stop_hooks_lib) + tests; write curated rules; add `docs-writer`/`docs-critic`; write `CLAUDE.md`, ADR-0001, this plan, TODO, session log.
5. **Verify:** `mkdocs build --strict` passes; hook tests pass.
6. **Ship:** create GitHub repo, commit atomically, push, enable Pages.
